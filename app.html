        How to Trust an Enterprise Certificate

        To run apps signed with an enterprise certificate, iOS requires you to trust the certificate manually.

        1. Open Settings on your device.
        2. Go to General → VPN & Device Management.
        3. Locate the relevant Enterprise App profile under "Enterprise App" or "Device Management".
        4. Tap the profile, then tap Trust and confirm your choice.
        5. Once trusted, you can open the app normally.

        Note:

        • Trusting the certificate ensures the app is authenticated and can run securely on your device.

        How to Fix "Unable to Verify App" or "Integrity Could Not Be Verified"

        This error appears when iOS cannot verify the app’s certificate. It usually means the certificate is revoked, expired, or your device is no longer allowed to use it.

        1. Basic Fixes

        1. Delete the app from your device.
        2. Restart your iPhone.
        3. Reinstall the app.

        2. Trust the App Manually

        1. Go to Settings → General → VPN & Device Management.
        2. Find the developer profile.
        3. Tap Trust.
        4. Open the app again.

        3. Check Internet & System Settings

        • Make sure you are connected to the internet on first launch.
        • Go to Settings → General → Date & Time and enable Set Automatically.

        4. Device Blacklisted (Common Case)

        Sometimes your device gets blacklisted from the certificate used to sign the app.

        • This means the app will never open, even if reinstalled.
        • This often happens with shared or public certificates.

        Fix options:

        • Install the app using a different certificate.
        • Wait until the app is re-signed with a new working certificate.

        5. Last Resort: Reset Device

        If your device is blacklisted and no new certificate is available:

        1. Backup your data.
        2. Go to Settings → General → Transfer or Reset iPhone → Erase All Content and Settings.
        3. Restore your backup.
        4. Reinstall the app.

        Important Notes

        • Factory reset is only needed in rare blacklist cases.
        • The issue is related to certificate validation, not your hardware.
        • Using a fresh certificate is usually the easiest fix.

        In most cases, switching to a new certificate or reinstalling the app resolves the issue.

        How to Enable Developer Mode on iOS

        Developer Mode is required to run sideloaded apps, test apps, or use Xcode on iOS 16 and later. Unlike older versions, the option is hidden by default until iOS detects a development-related action.

        1. Requirements

        • An iPhone running iOS 16 or newer
        • A sideloaded app, test app, or connection to a Mac with Xcode

        2. When Developer Mode Appears

        The toggle for Developer Mode usually becomes visible after one of these actions:

        • Installing a sideloaded app via AltStore, Sideloadly, or similar tools
        • Connecting your iPhone to a Mac running Xcode
        • Attempting to run a developer or test app

        3. Enabling Developer Mode

        1. Open Settings on your iPhone.
        2. Go to Privacy & Security.
        3. Scroll down and tap Developer Mode. If it isn’t visible, perform one of the actions above to trigger it.
        4. Toggle Developer Mode ON.
        5. Confirm by tapping Restart when prompted.
        6. After the device reboots, unlock your iPhone and tap Turn On to confirm.

        4. Why Developer Mode Is Needed

        • Allows execution of apps not from the App Store
        • Required for sideloaded apps installed via AltStore or Sideloadly
        • Required for running and testing apps with Xcode

        5. Notes & Tips

        • Developer Mode is hidden until a development-related trigger occurs — it’s normal if you don’t see it immediately.
        • Enabling it doesn’t permanently unlock the device or provide extra system features.
        • After enabling, sideloaded apps will work normally without additional prompts.

        6. Troubleshooting

        • If the toggle doesn’t appear, install a test or sideloaded app, or connect the device to Xcode once.
        • If enabling fails, ensure your iOS version is 16 or higher.

        Developer Mode is a safe, system feature designed to allow testing and sideloading while keeping casual users from accidentally enabling developer-only functionality.

        How to Allow a 3rd Party App Marketplace

        iOS requires you to approve and trust the developer profile before using third-party app marketplaces.

        1. When prompted during installation, tap OK or Allow to proceed.
        2. Open Settings → General → VPN & Device Management.
        3. Find the profile associated with the app store and tap Trust.
        4. Confirm your choice when prompted.
        5. Return to the website or app and tap Install App Marketplace if necessary.

        Note:

        • You need an internet connection for the app to verify the profile.
        • If the app doesn’t open immediately, try restarting your device.

        Once trusted, you can now use the third-party marketplace on your device.

        How to Create Your Own AltStore Source (Using AltStudio + GitHub Pages)

        This method uses AltStudio so you don’t need to manually write or understand JSON.

        1. What You Need

        • A GitHub account
        • Direct download links for your IPA files

        2. Create Your Source with AltStudio

        1. Go to https://altstudio.app
        2. Click Create Source
        3. Fill in your source info:
          • Name
          • Description
          • Icon URL
        4. Click Add App and fill in:
          • App Name
          • Bundle Identifier
          • Version
          • Download URL (your IPA link)
          • Icon URL
        5. Add as many apps as you want
        6. Click Export and download the generated file (usually apps.json)

        That’s it. AltStudio already builds the correct format for you.

        3. Upload the File to GitHub

        1. Create a new public repository
        2. Upload the exported file (apps.json) to the root of the repo

        4. Enable GitHub Pages

        1. Open your repository
        2. Go to Settings → Pages
        3. Select:
          • Branch: main
          • Folder: /root
        4. Save

        Your source will be available at:

        https://YOUR_USERNAME.github.io/REPO_NAME/apps.json

        5. Add Your Source to AltStore

        1. Open AltStore on your device
        2. Go to Browse → Sources
        3. Tap the + button
        4. Paste your GitHub Pages URL

        6. Hosting IPA Files

        Make sure your IPA links are direct downloads.

        Good options:

        • GitHub Releases (recommended)
        • Cloud storage providers

        Example:

        https://github.com/username/repo/releases/download/v1/app.ipa

        7. Tips

        • You don’t need to edit JSON manually when using AltStudio
        • Update apps by re-exporting and replacing the file
        • Always test your source after updating

        Done. You now have a working AltStore source without touching JSON.

        Finding AltStore Sources

        AltStore sources let you install apps from different repositories. Here’s how to find them, including official trusted ones.

        1. Official AltStore Trusted Sources

        The safest place to start is the official list of trusted sources maintained by AltStore.

        Official trusted sources page:

        https://faq.altstore.io/altstore-classic/trusted-sources

        These sources are reviewed and considered safe to use.

        2. AltStore Explore Page

        AltStore also provides a built-in discovery platform:

        https://explore.alt.store

        You can browse apps and sometimes find their sources directly from there.

        3. GitHub (Most Common)

        Most AltStore sources are hosted on GitHub Pages.

        Search for:

        • AltStore apps.json
        • AltStore source github

        Typical source format:

        https://username.github.io/repo/apps.json

        4. Community Sources

        You can also find sources shared by the community:

        • Reddit (r/AltStore)
        • Twitter / X
        • Discord servers

        These often include more apps, but are not always verified.

        5. Developer Sources

        Some app developers provide their own AltStore sources on their websites or GitHub pages.

        This is usually the best way to get official versions of specific apps.

        6. How to Add a Source

        1. Open AltStore
        2. Go to Sources
        3. Tap +
        4. Paste the source URL

        7. Safety Tips

        • Start with official trusted sources
        • Only use sources from known developers
        • Avoid random or suspicious links
        • Make sure apps install correctly

        Done

        You now know how to find safe and working AltStore sources.
